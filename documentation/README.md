# Documentation

This folder contains all project documentation, setup guides, and reference materials.

## Documents

### [Deployment Plan](./deployment-plan.md)
Complete guide for setting up development, pre-production, and production environments using GitHub and Vercel. Includes step-by-step instructions, checklists, and workflow documentation.

### [API Documentation](./api-documentation.md)
Detailed documentation of all API endpoints, including request/response formats, error handling, and external service integrations.

### [Environment Variables](./environment-variables.md)
Comprehensive list of all required environment variables, setup instructions for different environments, and security best practices.

### [Architecture Analysis](./architecture-analysis.md)
Detailed analysis of the application architecture, recommended patterns, and migration plan for scalable development with external integrations.

### [Lessons Learned](./LESSONS_LEARNED.md)
Key learnings from optimization work, bug fixes, and architectural decisions. Includes patterns, pitfalls, and best practices discovered during development.

### [Filtering Architecture](./FILTERING_ARCHITECTURE.md)
Problem analysis and solution design for the Discover page filtering system. Documents the issues with current implementation and provides a detailed implementation plan using nuqs and Suspense boundaries.

## Adding New Documentation

When adding new documentation files:

1. Use descriptive filenames with lowercase and hyphens (e.g., `feature-guide.md`)
2. Update this README with a link to the new document
3. Follow markdown formatting conventions
4. Include a brief description in the document list above
